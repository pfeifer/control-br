# XKore 2 / Configurações de proxy
# [conf] XKore_listenIp <IP> [info] Se esta opção for definida, o servidor do XKore Proxy atenderá a qualquer pedido de conexão de qualquer adaptador de rede instalado no sistema (recomendado). Caso tenha múltiplos adaptadores de rede no seu sistema e queira restringir o acesso ao servidor XKore Proxy, defina esta opção para o IP do adaptador de rede desejado.
XKore_listenIp 127.0.0.1
# [conf] XKore_listenPort <porta> [info] O número da porta que o client do jogo deverá usar para conectar-se ao servidor XKore 2 ou Proxy. Se esta opção não for definida, o XKore 2 e o XKore Proxy usarão as portas 6900 e 6901, por padrão.
XKore_listenPort 6901
# [conf] XKore_publicIp <IP> [info] Endereço de rede para uso do XKore 2. Use 0.0.0.0 para tornar todas os adaptadores disponíveis. Por padrão, usa 127.0.0.1 ou se preferir localhost, para que apenas clients no mesmo sistema possam se conectar. [obs] O IP interno é diferente do IP externo: para descobrir o seu IP interno use o comando ipconfig /all (Windows) ou ifconfig (Linux); para descobrir o IP externo, visite http://www.whatismyip.com/
XKore_publicIp 127.0.0.1
# [conf] XKore_ID <servidor> [info] Defina esta opção para o nome do servidor de jogo (exemplo: Chaos, Loki ou Sarah). Este nome aparecerá no client do jogo ao conectar-se ao XKore 2 para escolher um servidor. [obs] Este valor não precisa ser o nome do servidor do jogo. Contudo, não se esqueça que o client do jogo pode usar este valor, por exemplo, para dar nome às capturas de tela que você fizer.
XKore_ID

######## Configuração principal ########

# [conf] alias_<nome_alternativo> <comando_do_console> [info] Esta opção permite que você defina um nome alternativo, um "apelido", para comandos de console. Exemplo: https://openkorebrasilwiki.miraheze.org/wiki/Alias
alias_heal sp 28

# [conf] allowedMaps <mapas> [info] Limita os mapas que o Kore pode acessar. Caso definida, esta opção fará com que ele reaja de acordo com allowedMaps_reaction. Para fornecer múltiplos mapas, separe-os por vírgula. Exemplo: allowedMaps prontera, prt_in, prt_fild08, moc_fild01
allowedMaps
# [conf] allowedMaps_reaction <valor> [info] Atitude a ser tomada quando o personagem não se encontra na lista de mapas fornecida por allowedMaps. [val] 0 - Respawn (Reaparece) usa Teleporte nível 2. 1 - Desconecta-se e finaliza. [obs] Definido para 0 é necessário ter a habilidade Teleporte ou o item Asas de Borboleta.
allowedMaps_reaction 1

# [conf] attackAuto <valor> [info] Determina o comportamento de ataque. [val] -1 - Não ataca nenhum monstro, nunca. Desativa a AI de ataque. 0 - Apenas revida um ataque recebido. 1 - Ataca apenas para se defender, defender seu Master ou seu grupo. 2 - Ataca monstros de forma agressiva, procurando ativamente por um alvo. [obs] Apenas surte efeito caso a AI esteja no modo automático. Caso nenhum alvo seja encontrado, o tempo de espera até a próxima checagem é de ai_attack_auto <segundos> (timeouts.txt). O OpenKore não faz Kill Steal (KS) nativamente.
attackAuto 2
# [conf] attackAuto_party <valor> [info] Esta opção define se o Kore deve atacar monstros que estão atacando membros do grupo. [val] 0 ou vazio - Não fazer nada. 1 - Ataque monstros, que estão lutando contra membro(s) do grupo. 2 - Ataque monstros apenas se eles começaram a atacar membro(s) do grupo. [obs] Use valor 2 se você estiver sendo pressionado para impedir que você ataque-o primeiro, enquanto ainda está se movendo em direção ao Slave/Master. A menos que esses monstros sejam conhecidos por terem o hábito de mudar de alvo, independentemente de quem fez o primeiro ataque, esse valor garante que os monstros sempre atacarão o Slave/Master. Esta opção funciona de forma independente do attackAuto, portanto, se esta opção estiver habilitada, você irá atacar monstros envolvidos em batalha com seu grupo independentemente de como o attackAuto está configurado. O Kore só irá atacar monstros comprometidos com o alvo seguinte se não houverem outros monstros agressivos atacando ele próprio. Se você deseja dar prioridade aos monstros envolvidos por membro(s) do grupo, defina attackAuto para 0. Só tem efeito quando o AI está no modo automático.
attackAuto_party 1
# [conf] attackAuto_onlyWhenSafe <booleano> [info] 1 - Desativa o ataque agressivo quando não for seguro. [obs] É considerado "seguro" quando não há jogadores nas proximidades, exceto do seu grupo.
attackAuto_onlyWhenSafe 0
# [conf] attackAuto_followTarget <booleano> [info] Quando o Kore está seguindo um personagem e esta opção estiver ativada, ele irá atacar os monstros em combate com o Master. [obs] Esta opção funcionará independentemente do attackAuto (a menos que este esteja definido como -1). Ele dará prioridade em atacar monstros que estão o atacando (Actor::You) antes de atacar o monstro em combate com o Master, caso o attackAuto permita. Apenas surte efeito caso a AI esteja no modo automático.
attackAuto_followTarget 1
# [conf] attackAuto_inLockOnly <número> [info] Esta opção configura o comportamento de ataque fora do lockMap. [val] 0 - Ataca monstros em qualquer mapa. 1 - Ataca monstros no lockMap e no caminho até lá. >= 2 - Ataca monstros apenas no lockMap. [obs] Esta opção é dependente do attackAuto e do mon_control.txt.
attackAuto_inLockOnly 1
# [conf] attackAuto_notInTown <booleano> [info] 1 - Desativa o ataque automático dentro de cidades.
attackAuto_notInTown 1
# [conf] attackDistance <valor> [info] Esta opção define a distância, em número de células, da qual monstro-alvo será atacado. Este valor também é o valor padrão caso o dist (definido em attackSkillSlot, attackComboSlot ou partySkill) não seja especificado nos blocos que aceitam condições recíprocas. [obs] Recomendável para melee (atacante corpo-a-corpo) usar attackDistance 1. https://openkorebrasilwiki.miraheze.org/wiki/Categoria:Condi%C3%A7%C3%B5es_Rec%C3%ADprocas
attackDistance 1.5
# [conf] attackDistanceAuto <booleano> [info] 1 - O Kore irá auto definir o attackDistance e o attackMaxDistance. [obs] Recomendável para melee habilitar esta opção.
attackDistanceAuto 0
# [conf] attackMaxDistance <número> [info] Esta opção especifica a distância máxima, em células, entre o Kore e o monstro-alvo. Se a distância se tornar maior que a especificada, ele tentará se aproximar. [obs] Recomendável para melee usar attackMaxDistance 1.
attackMaxDistance 2.5
# [conf] attackMaxRouteDistance <número> [info] Define a distância máxima de rota para atingir um alvo. Esta configuração é útil para evitar que o Kore circunde longas escarpas/paredes apenas para atacar um monstro. [obs] Útil quando está em mapas que contém labirintos e detecta um monstro que está atrás de uma parede.
attackMaxRouteDistance 100
# [conf] attackMaxRouteTime <segundos> [info] Tempo máximo que o Kore gastará para calcular uma rota para chegar até o alvo.
attackMaxRouteTime 4
# [conf] attackMin<Player/Portal>Distance <número> [info] O Kore não vai atacar monstros não-agressivos que estão dentro da distância especificada em número de blocos de um(ns) jogador(es)/portal no mapa. [obs] Para "Player" use distância 6 ou maior para evitar KS. Para "Portal" use 7 ou maior, para evitar que o Kore saia do lockMap, se esta opção não estiver definida, ele usará um valor padrão 4.
attackMinPlayerDistance 2
attackMinPortalDistance 4
# [conf] attackUseWeapon <booleano> [info] Se esta opção for definida como 1 e o Kore não estiver usando habilidades para atacar, ele será autorizado a usar sua arma para atacar. Caso contrário for definido com 0 (quando desabilitada, ele não se moverá até o inimigo para atacar e usará skills conforme configuradas incluindo distância), ele nunca se moverá em direção ao alvo para atacar com sua arma. A maioria magos (exemplo, Sacerdotes utilizados como Slave) recomenda-se que esta opção seja definida com 0, a não ser que a classe seja do tipo battle.
attackUseWeapon 1
# [conf] attackNoGiveup <booleano> [info] 1 - O Kore não vai parar de atacar o alvo, mesmo que não cause danos. [obs] Isso é útil se o seu attackSkillSlot é uma habilidade não prejudicial, por exemplo Spell Breaker.
attackNoGiveup 0
# [conf] attackCanSnipe <booleano> [info] Defina esta opção para 1 se o seu personagem é capaz de atacar a distancia (Arqueiros/Magos). O Kore então tentará atacar através de paredes ou desníveis, em vez de andar ao redor deles.
attackCanSnipe 0
# [conf] attackCheckLOS <booleano> [info] Esta opção alterna o uso do código de verificação da Linha de visão (LOS - Line Of Sight) ao atacar. Se esta opção estiver habilitada e você for um atacante à distância (ou seja, attackDistance for definido como um valor maior que 2), isso irá verificar se você tem uma linha de visão clara para o alvo, se não, ele tentará mover para um espaço onde consiga atacar, respeitando runFromTarget_dist e followDistanceMax.
attackCheckLOS 0
# [conf] attackLooters <booleano> [info] Se esta opção é definida como 1 e um item no chão desapareceu, o Kore atacará automaticamente monstros diretamente no mesmo local onde o item foi visto pela última vez (atacará todos monstros que roubarem seu loot). [obs] Isso só funcionará se o Kore estiver configurado para pegar o item que desapareceu então provavelmente, você tem que configurar como apanha-los e como irá recolhê-los (item_pickup.txt). Não tem efeito quando sentar automaticamente estiver ativado.
attackLooters 0
# [conf] attackChangeTarget <booleano> [info] 1 - O Kore mudará seu alvo caso haja um monstro mais agressivo.
attackChangeTarget 1
# [conf] aggressiveAntiKS <booleano> [info] 1 - Esta opção fará com que o Kore desista de um alvo se um terceiro atacá-lo e ainda não causou dano, esteja o monstro atacando-nos ou não. [obs] Ele não irá desistir de um monstro se já tiver causado dano e, portanto, seu bot está sofrendo KS.
aggressiveAntiKS 0

autoSpell_safe

# [conf] avoidGM_namePattern <expreção_regular> [info] Por padrão, o Kore identifica os Game Masters como aqueles cujos nomes seguem o padrão: /^([a-z]?ro)?-?(Sub)?-?\[?GM\]?/ [obs] Não inclua as barras oblíquas ("/"). http://openkore.com/index.php/AvoidGM_namePattern
avoidGM_namePattern
# [conf] avoidGM_near <valor> [info] Quando o Kore encontra jogadores cujo nome corresponde a um determinado padrão para os nomes de Game Master (avoidGM_namePattern), ele pode optar por evitá-los quando forem detectados. [val] 0 - Não evite GMs. 1 - Teleporte p/ longe e desconecte. 2 - Desconecte imediatamente. 3 - Teleporte p/ longe. 4 - Respawn (Renascer). [obs] Se o Kore se desconectar para evitar um GM, reconectará depois que o número de segundos especificado em avoidGM_reconnect tenha decorrido.
avoidGM_near 0
# [conf] avoidGM_near_inTown <booleano> [info] 1 - O Kore tentará evitar GMs mesmo quando estiver na cidade. 0 - Desabilita. [obs] avoidGM_near deve ser configurado.

avoidGM_talk 0
